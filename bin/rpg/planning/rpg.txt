------------------------------------------------------------
                      <<abstract>>
                     GameCharacter
------------------------------------------------------------
  - name          : String
  - health        : int
  - maxHealth     : int
  - attackPower   : int
  - defense       : int
------------------------------------------------------------
  + getName()           : String
  + getHealth()         : int
  + takeDamage(int)     : void
  + isAlive()           : boolean
  + heal(int)           : void
  + abstract attack()   : int
------------------------------------------------------------


============================================================
  INHERITANCE  (extends GameCharacter)
============================================================

        GameCharacter
            |
     -------+-------
     |             |
   Player        Enemy


------------------------------------------------------------
                       Player
------------------------------------------------------------
  - inventory       : HealingItem[3]
  - gold            : int
  - enemiesDefeated : int
------------------------------------------------------------
  + attack()            : int
  + useItem(int)        : void
  + addItem(HealingItem): boolean
  + hasItems()          : boolean
------------------------------------------------------------
  Notes:
    - Choosing attack() or useItem() ends the player's turn.
    - useItem(int slot) uses the item in the specified
      inventory slot (0-2).
    - addItem() returns false if inventory is full.
------------------------------------------------------------


------------------------------------------------------------
                        Enemy
------------------------------------------------------------
  - enemyType     : EnemyType
  - dropChance    : float         // base 0.30 (30%)
------------------------------------------------------------
  + attack()                : int
  + rollDrop()              : HealingItem
  + getEnemyType()          : EnemyType
------------------------------------------------------------
  Notes:
    - rollDrop() generates a random value 0–1. If the
      value is <= dropChance, a HealingItem is returned.
      Otherwise returns null.
    - attack() returns damage based on attackPower.
------------------------------------------------------------


------------------------------------------------------------
                      MiniBoss
------------------------------------------------------------
  - phase         : int
------------------------------------------------------------
  + attack()                : int
  + phaseTransition()       : void
------------------------------------------------------------
  Notes:
    - Extends Enemy with higher stats.
    - Spawned by EnemySpawner after a set number of
      enemy encounters.
    - phaseTransition() can escalate stats or behavior
      at certain health thresholds (e.g., 50% HP).
------------------------------------------------------------


============================================================
  STANDALONE CLASSES
============================================================

------------------------------------------------------------
                    HealingItem
------------------------------------------------------------
  - name          : String
  - description   : String
  - healAmount    : int
------------------------------------------------------------
  + use(GameCharacter)  : void
  + getHealAmount()     : int
  + getName()           : String
  + getDescription()    : String
------------------------------------------------------------
  Notes:
    - Consumed from the player's inventory on use.
    - Using a HealingItem ends the player's turn.
------------------------------------------------------------


------------------------------------------------------------
                    EnemySpawner
------------------------------------------------------------
  - enemyTypes    : List<EnemyType>
  - spawnCount    : int
  - bossTrigger   : int
------------------------------------------------------------
  + spawnEnemy()        : Enemy
  + shouldSpawnBoss()   : boolean
  + resetCount()        : void
------------------------------------------------------------
  Notes:
    - spawnEnemy() picks a random EnemyType and creates
      an Enemy with stats based on that type.
    - spawnCount increments on each spawn. Once it
      reaches bossTrigger, shouldSpawnBoss() returns
      true and a MiniBoss is spawned instead.
    - resetCount() resets spawnCount after boss is
      defeated.
------------------------------------------------------------


------------------------------------------------------------
                    BattleManager
------------------------------------------------------------
  - player        : Player
  - currentEnemy  : Enemy
  - spawner       : EnemySpawner
  - turnCount     : int
------------------------------------------------------------
  + startBattle()       : void
  + playerTurn(String)  : void
  + enemyTurn()         : void
  + checkVictory()      : boolean
  + awardDrops()        : void
  + endBattle()         : void
------------------------------------------------------------
  Notes:
    - Orchestrates the combat loop.
    - playerTurn(action) takes "attack" or "use X" where
      X is inventory slot number.
    - Turn sequence: player acts → check if enemy dead →
      enemy acts → check if player dead → repeat.
    - awardDrops() calls enemy's rollDrop() on victory
      and adds the result to player's inventory if the
      roll succeeds and inventory has space.
------------------------------------------------------------

============================================================
  KEY SIMPLIFICATIONS FROM ORIGINAL DESIGN
============================================================

  ✓ Removed defend action - player only attacks or uses items
  ✓ Removed charged attacks - all enemies do regular attacks
  ✓ Consolidated BasicEnemy and ChargedEnemy into single
    Enemy class
  ✓ Simplified turn logic - no charge warnings or defend
    mechanics to track
  ✓ MiniBoss is just a stronger Enemy, not a special type

  This reduces code complexity by ~40% while maintaining:
    - Turn-based combat
    - Inventory management
    - Random enemy encounters
    - Boss fights
    - Item drops and rewards
